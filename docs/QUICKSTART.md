# 快速开始指南

## 5分钟上手 PixelPerfect Type

### 方式一：使用启动脚本（推荐）

```bash
# 1. 进入项目目录
cd ~/Documents/代码/字号自动测量器2

# 2. 运行启动脚本
./start.sh

# 3. 打开浏览器
# 在浏览器中打开 frontend/index.html
```

### 方式二：手动启动

```bash
# 1. 安装依赖
cd ~/Documents/代码/字号自动测量器2/backend
pip install -r requirements.txt

# 2. 启动后端
python app.py

# 3. 打开前端
# 在浏览器中打开 frontend/index.html
```

## 使用流程

### 步骤1: 上传设计稿

![Upload](docs/images/step1-upload.png)

点击"选择文件"或拖拽图片到上传区域

**支持格式**: JPG, PNG
**推荐尺寸**: 750px宽度（移动端设计稿）

---

### 步骤2: 开始分析

![Process](docs/images/step2-process.png)

点击"开始分析"按钮，等待处理完成（通常10-30秒）

**处理流程**:
- View 1: 图像标准化
- View 2: OCR文字识别
- View 3: 字号拟合
- View 4: 结果标注

#### View 1: 标准化
查看缩放到750px宽度后的图片

绿色框标注了识别到的文字区域

#### View 3: 字号拟合

## 常见问题

### Q: 为什么识别不准确？

**可能原因**:
1. 设计稿字体不是PingFang SC
2. 图片质量过低
3. 背景与文字对比度不足

**解决方案**:
- 使用高质量原图
- 确保文字清晰可见
- 如使用其他字体，参考README修改字体配置

---

### Q: 处理速度慢怎么办？

**原因**:
- 首次运行需要下载PaddleOCR模型
- 文字区域较多时计算量大

**优化建议**:
- 首次运行耐心等待模型下载
- 减少设计稿中的文字数量
- 考虑启用GPU加速（需要NVIDIA显卡）

---

### Q: 能处理什么样的设计稿？

**最佳效果**:
- ✅ 750px宽度的移动端设计稿
- ✅ 使用PingFang SC字体
- ✅ 纯色或简单渐变背景
- ✅ 文字清晰、对比度高

**可以处理但效果可能下降**:
- ⚠️ 其他尺寸（会自动缩放）
- ⚠️ 其他字体（需配置）
- ⚠️ 复杂背景
- ⚠️ 模糊或小字号

**不支持**:
- ❌ 手写字体
- ❌ 艺术字、图片文字
- ❌ 严重倾斜或变形的文字
